## 🎛️ Configurações Aplicadas

### Aplicação Quarkus (`application.properties`)

✅ **Logging**
- JSON estruturado habilitado
- Pretty-print em desenvolvimento
- Campos customizados (service.name, environment)

✅ **OpenTelemetry**
- Endpoint: http://localhost:4317
- Traces: 100% em dev, 10% em prod
- Logs e métricas habilitados
- Propagação de contexto (tracecontext, baggage)

✅ **Micrometer/Prometheus**
- Métricas em `/q/metrics`
- Binders JVM, System, HTTP habilitados

✅ **Context Propagation**
- MDC integrado com ARC

### Prometheus (`prometheus.yml`)

✅ **Targets configurados**
- Prometheus self-monitoring
- Quarkus app (host.docker.internal:8080)
- OpenTelemetry Collector
- Elasticsearch (opcional)
- Grafana

✅ **Scrape interval: 15s**

### OpenTelemetry Collector

✅ **Receivers**
- OTLP gRPC (4317)
- OTLP HTTP (4318)
- Prometheus metrics

✅ **Processors**
- Batch processing
- Resource attributes
- Sensitive data filtering (password, token, apikey)

✅ **Exporters**
- Console (debug)
- Prometheus (métricas)
- Elasticsearch (logs, traces, metrics)

### Grafana

✅ **Datasources pré-configurados**
- Prometheus (métricas) - default
- Elasticsearch (logs)
- Elasticsearch-Traces (traces)
- Elasticsearch-Metrics (métricas OTel)

✅ **Dashboard**
- Quarkus Application Overview
  - HTTP Request Rate
  - HTTP Request Latency (P95)
  - JVM Memory Usage
  - HTTP Error Rate
  - Stats (Total Requests, Active Threads, CPU, Uptime)

## 🚀 Como Usar

### Opção 1: Scripts Automatizados (Recomendado)

**Windows:**
```powershell
.\start-observability.ps1
```

**Linux/macOS:**
```bash
chmod +x start-observability.sh
./start-observability.sh
```

### Opção 2: Docker Compose Manual

```bash
docker-compose up -d
```

### Iniciar a Aplicação

```bash
cd logging-quarkus
./mvnw quarkus:dev
```

### Acessar Dashboards

1. **Grafana**: http://localhost:3000 (admin/admin)
2. **Prometheus**: http://localhost:9090
3. **Kibana**: http://localhost:5601
4. **Graylog**: http://localhost:9000 (admin/admin)

## 🔧 Comandos de Manutenção

### Ver Logs dos Serviços

```bash
# Todos os serviços
docker-compose logs -f

# Serviço específico
docker-compose logs -f grafana
docker-compose logs -f prometheus
docker-compose logs -f elasticsearch
```

### Reiniciar Serviços

```bash
# Todos
docker-compose restart

# Específico
docker-compose restart grafana
```

### Limpar e Recomeçar

```bash
# Parar tudo
docker-compose down

# Parar e remover volumes (CUIDADO: apaga dados)
docker-compose down -v

# Recomeçar do zero
docker-compose up -d
```

### Verificar Recursos

```bash
# Uso de espaço
docker system df

# Limpar recursos não utilizados
docker system prune
```
